УРА ДОКЕР

- Контейнеризация. Изолированный запуск приложения в контейнерах, удобнее чем ВМ. Легче запускать и развёртывать, круто.
- Что такое контейнер?
	- Аппаратная виртуализация - создание ВМ. Большой сервер с кучей ресурсов. Использует гипервизор для распределения ресурсов.
	- Контейнерная виртуализация - запуск конкретной программы в изолированной среде. Работает поверх ОС, то есть юзает меньше ресурсов и проще в управлении.
- Из чего состоит Docker?
	- Три механизма ядра, что использует Docker:
		- Namespaces. Изолированное пространство имён для процессов и т.п.
		- cGroups. Выделение конкретного количества ресурсов для контейнера.
		- UnionFS. Общая файловая система.
			- Файловая система делится на образы, что наслаиваются друг на друга и объединяются в единый слой.
- Docker Engine состоит из:
	- Образов
	- Контейнеров
	- Volumes
	- Networks
	- Работает через Docker daemon и CLI клиента, что отправляет запросы через REST API слой.
- Образ:
	- Набор команд
	- Каждая команда - новый слой
	- Слои совместно используются в разных образах в силу своей неизменяемости
- Контейнер:
	- Запущенный образ
	- Когда контейнер удаляется - удаляется слой для записи
- Dockerfile
	- Файл, где описываются инструкции создания образа
- Multistage build
	- Приём для уменьшения размера образа
	- Используем несколько инструкций FROM. Артефакты оттуда передаём через COPY в финальный слой
- Docker registry
	- Место откуда берутся образы
	- Ищётся локальный образ. Если он отсутствует, то скачивается из DockerHub, либо из корпоративных ссылок
- Docker volumes
	- Прокинуть / примонтировать файлы, чтобы не потерять их после остановки контейнера.
	- Полезно для БД, конфигов и т.п.
	- Типы volumes:
		- bind-mount - тоннель междй файлами на пк и внутри контейнера. Удобно для конфигов
		- volume - вместо монтирования кокнретной точки, монтируем это всё в специальное docker окружение хоста. Физически не видим где и как расположены файлы, из-за чего не можем их трогать. Используем для БД
		- tmpfs-mount - монтирование в ОЗУ машины. Более быстрый доступ к данным.
- Docker networking
	- Bridge - режим по умолчанию. Создаётся частная подсеть на машине. Конйтенеры внутри неё общаются без проблем.
	- None - никакого взаимодействия с сетью
	- Host - используется подсеть самой машины. Если нас не интересует сетевая изоляция
- Установка
	- Ну там в зависимости от ОС. На винде прослойка в виде WSL
- Основные команды
	- docker ps
	- docker run 
	- docker start
	- docker stop
	- docker images
- docker-compose
	- Инструмент для оркестрации нескольких контейнеров
	- Описываем инструкции в файле, сколько контейнеров, где, как, их параметры и т.д.
	- Подходит под локальную разработку
	- Основные команды:
		- docker-compose up
		- docker-compose down
		- docker-compose restart
		- docker-compose ps
		- docker compose build
		- docker compose logs
- Docker Swarm
	- Продвинутый оркестратор встроенный в Docker
	- Подходит для продакшена
	- Мастер нода с контейнерами менеджерами, и рабочая нода с контейнерами для работы
	- Можно оркестрировать удалённые приложения
- Kubernetes
	- Преподаётся на маге кстати, манит сучка
	- Очень крутая продвинутая штука
