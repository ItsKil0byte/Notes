- **Фреймворки, организация и устройства.**
- Фреймворки:
	- Определение - заготовка для кода. 
	- Задаёт правила архитектуры для будущего проекта. Собственно говоря - его единственное назначение.
	- Стандартная библиотека для реализации тех или иных задач - хелперы.
	- Организация файловой системы. Необязательное, но крайне хорошее свойство.
- Из каких частей состоит:
	- Ядро. Взаимодействие всех основных частей фреймворков и стандартных интерфейсов.
	- Конфигурация.
	- DI / Container.
		- Ленивая загрузка. Ресурсы загружаются только при необходимости.
		- DI - автоматически создаются необходимые объекты для работы приложения.
	- Router. Какой код будет обрабатывать тот или иной запрос.
		- Можно перенести ответственность по ответу для пользователя на веб сервер.
	- Middleware. Промежуточный слой с полезным действием.
	- Модули и контроллеры. Контроллер хотим иметь как можно более худым.
	- Сервисы и репозитории. Также валидаторы данных.
	- Шаблонизаторы. Преобразование ответа в нужный формат.
	- Логи.
	- Обработка ошибок.
